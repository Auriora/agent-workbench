# Agent Workbench v0.4.0

## Highlights

- Added the `awb` developer CLI for common repository maintenance workflows,
  including package checks, local installs, plugin refresh/status, MCP smoke
  checks, cache inspection, spec lifecycle wrappers, release preflight, version
  bumping, and GitHub release publishing.
- Hardened package and plugin installation across Codex, Claude Code, and Kiro,
  including GitHub release tarball install guidance and Node 24 native rebuild
  documentation.
- Improved workspace watcher behavior with ignore-aware path policy,
  asynchronous queue processing, and status freshness reporting.
- Strengthened MCP runtime contracts with repo-root authority and consistent
  error envelope handling.

## Added

- `awb release preflight`, `awb release bump-version`, and
  `awb release github` commands for release preparation and GitHub release
  publishing.
- Repository-owned CLI commands for validation, packaging, plugin operations,
  MCP smoke checks, cache inspection, spec lifecycle checks, and diagnostics.
- Agent Skills validation and CI coverage for checked-in Agent Workbench skills.
- Shared path policy and workspace safety behavior used across file scanning,
  docs queries, and hook feedback paths.
- Workspace watcher queue processing and runtime status freshness surfaces.

## Changed

- Package install documentation now points users at GitHub release tarballs
  instead of npm registry publishing.
- Codex hook installation now uses absolute package paths and safer default
  feedback behavior.
- Plugin launch materialization for Codex MCP setup was corrected so packaged
  launchers resolve the repository root and package root reliably.
- Startup graph warmup is serialized to reduce runtime contention during launch.
- Integration and packaging validation were expanded for cross-platform package,
  hook, launch, and native dependency scenarios.
- Project files, scripts, docs, and source files now carry GPL/Auriora licensing
  headers.

## Fixed

- Fixed Codex plugin-root launch behavior for packaged MCP startup.
- Fixed Codex hook feedback defaults and hook installation path handling.
- Fixed repository-root authority handling in MCP resources and tools.
- Fixed MCP error envelope consistency so failures use the shared response
  shape.

## Documentation

- Added and closed implementation records for cross-platform packaging,
  developer CLI workflow tools, repo-root authority, MCP error envelopes, shared
  path policy, watcher ignore synchronization, and related lifecycle work.
- Updated plugin, install, native dependency, architecture, runtime contract,
  workspace safety, and lifecycle documentation to reflect the delivered
  behavior.
- Added durable backlog and closure records for completed specs and follow-up
  work.

## Validation

- Release preflight and package dry-run checks were added to the repository
  workflow.
- CI/plugin validation now covers package manifests, plugin manifests, hooks,
  skills, launch files, and install metadata.
- Fixture-backed tests were added or updated for workspace watcher behavior,
  path policy consistency, MCP error envelopes, repo-root authority, Agent
  Skills validation, dev CLI workflows, hook installation, and package launch
  behavior.

## Known Issues

- GitHub PR metadata is not part of release-note generation in this release.
- Some follow-up package distribution work remains routed through the backlog,
  including broader marketplace and turnkey native dependency improvements.

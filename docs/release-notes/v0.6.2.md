# Agent Workbench v0.6.2

## Highlights

- Makes stale or invalid repository snapshots recoverable through one
  daemon-owned refresh flow shared by connected coding-agent clients.
- Rebuilds reference and documentation discovery around explicit completeness,
  ranking, readiness, and trust receipts instead of success-shaped partial
  results.
- Aligns Claude Code, Codex, and Kiro session-start behavior around the same
  core hook construction while preserving client-specific launch adapters.

## Added

- Added an agent-callable refresh executor that publishes a replacement
  snapshot and lets status report convergence, blockers, and recovery actions.
- Added snapshot path-validity evidence for deleted and inaccessible indexed
  paths, including orientation reuse and refresh decisions.
- Added provider-aware integration health for Codex, Claude Code, and Kiro.
- Added complete ranked documentation universes with frozen cursor pagination,
  authority-map ownership, ranking reasons, corpus counters, and readiness
  state tied to the selected snapshot.
- Added parser-route reference completeness receipts that disclose route
  exhaustion, result-count basis, policy exclusions, and unresolved searchable
  candidates.

## Changed

- `verification_plan` now selects all repository CI gates for plugin changes,
  ranking integration-specific validation before generic typecheck and tests.
- `context_for_task` now respects explicitly named Claude Code, Codex, and Kiro
  integration scope and ranks the first-party hook modules and sync script.
- Session-start hook payload construction now shares one core behavior across
  the three packaged integrations.
- Orientation and status now agree on documentation-ranking readiness and do
  not present a blocked ranking route as reusable first-read context.
- Ranked-documentation owner classification and graph-store reads are bounded;
  oversized canonical documents no longer require unbounded whole-file reads.
- The README and install runbooks now present user and coding-agent workflows,
  including Windows PowerShell and updateable marketplace installation paths.

## Fixed

- Fixed stale deleted paths crashing `find_references`; blocked graph results
  now return structured evidence and a callable recovery path.
- Fixed reference results that mixed prose mentions, unrelated same-name
  symbols, and incomplete catalog scans into apparently complete output.
- Fixed fabricated `impact` node IDs being indistinguishable from real symbols
  with no callers.
- Fixed secret-path diagnostics reporting policy exclusions as missing files.
- Fixed sensitive traversal and host-path values being redacted in source text
  but exposed in symbol signatures.
- Fixed same-schema patch upgrades leaving a dead daemon owner and unpublished
  build state that prevented MCP initialization.
- Fixed interactive refresh publication being delayed by derived-store
  maintenance and repeated global FTS cleanup.
- Fixed documentation ranking readiness races, legacy-universe migration,
  continuation re-admission, and typed environment-failure presentation.

## Packaging And Installation

- Package, Codex plugin, Claude Code plugin, server card, and distribution
  manifest versions are synchronized at `0.6.2`.
- Repository release notes retain their structured frontmatter, while the
  GitHub release workflow publishes a clean Markdown body without that metadata
  block.
- The package continues to ship the `agent-workbench-local` marketplace for
  clone-free Codex and Claude Code installation.
- Repository-local Codex and Claude Code launchers remain available for checkout
  testing, while user installs launch the globally installed package runtime.

## Documentation

- Promoted snapshot validity, refresh convergence, reference completeness,
  authority-aware ranking, ranking readiness, and trust behavior into the
  durable runtime, MCP, graph-store, integration, and proof documentation.
- Closed and removed Specs 039 through 044 after durable promotion and recorded
  their final implementation and cleanup commits.
- Added installed-client and dogfood evidence for both successful recovery and
  the remaining product gaps.

## Validation

- The final release candidate passed 1,067 Vitest tests across 100 files, plus
  TypeScript typecheck, plugin validation, skill validation, developer CLI
  tests, package preflight, and package dry-run inspection.
- Installed Codex and Claude Code sessions selected the same published fresh
  snapshot and exercised the packaged `0.6.1` candidate that this release
  supersedes.
- The release workflow reruns package, integration, plugin, skill, typecheck,
  and full-suite gates before publishing the release artifact.

## Known Issues

- EB059 retains ranked-universe capacity, eviction, cursor-staleness, and
  detailed observability policy.
- EB061 retains disclosure or extraction of dynamic-import member-call coverage
  outside the current parser reference route.
- EB062 retains duplicate node-FTS identity normalization within one extraction
  batch.
- EB063 retains shared redaction of arbitrary generic provider error messages.
- EB064 will isolate fixture documentation from production repository ranking
  and ensure a governing owner precedes draft/supporting mention-heavy evidence.
- EB065 will compact routine `verification_plan` skipped-path records into
  bounded grouped summaries.
